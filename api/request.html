Can\Server\Request::setCookie(string $name[, string $value[, int $expire = 0[, string $path = '/'[, string $domain[, bool $secure = false[, bool $httponly = false]]]]]], ))¶

Set the cookie. Actually this method adds Set-Cookie response header with provided options.

Parameters:
  • $name (string) – The name of the cookie.
  • $value (string) – The value of the cookie.
  • $expire (int) – The time the cookie expires. This is a Unix timestamp so is in number of seconds since the epoch.
  • $path (string) – The path on the server in which the cookie will be available on.
  • $domain (string) – The domain that the cookie is available to.
  • $secure (bool) – Indicates that the cookie should only be transmitted over a secure HTTPS connection from the client.
  • $httponly (bool) – When true the cookie will be made accessible only through the HTTP protocol.
Throws :
  • InvalidParametersException If invalid parameters are passed to the method.
Returns:

boolean true on success, false on failure
Can\Server\Request::sendFile(string $filename[, string $root[, string $mimetype[, mixed $download[, int $chunksize=10240]]]])¶

Serve file in a safe and convenient way. See Routing Static Files for detailed information.

Parameters:
  • $filename (string) – Name of the file to send.
  • $root (string) – Root directory where file $filename to be expected.
  • $mimetype (string) – Add this mimetype instead of automatically guesed one.
  • $download (bool|string) – Force download of the file. If true, the filename will be determine automatically, if string, the filename will be set to value of $download.
  • $chunksize (int) – The size of the chunks if chunked transfer encoding is used (Serving of files with filesize >= $chunksize). Default value is 8192 bytes.
